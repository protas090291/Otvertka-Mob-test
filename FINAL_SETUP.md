# ✅ Финальная настройка для Time Web Cloud

## ✅ Что было изменено

Теперь используется только Metro Bundler на порту 8080, без Express сервера.

## 📋 Что нужно сделать в Time Web Cloud

### 1. Изменить команду запуска

В разделе "Настройки деплоя" → "Команда запуска":

**Измените на:**
```
npx expo start --host lan --port 8080 --no-dev --minify
```

### 2. Обновить переменные окружения

В разделе "Настройки" → "Переменные":

**УДАЛИТЕ:**
- `METRO_PORT` (больше не нужна)

**ОСТАВЬТЕ:**
- `PORT` = `8080`
- `NODE_ENV` = `production`
- `EXPO_NO_DOTENJ` = `1`
- `EXPO_DEVTOOLS_LISTEN_ADDRESS` = `0.0.0.0`
- `NODE_OPTIONS` = `--max-old-space-size=4096`

### 3. Health Check Path

Оставьте:
```
/status
```

Metro Bundler предоставляет health check через `/status`.

### 4. Сохраните и перезапустите

1. Нажмите "Сохранить данные"
2. Перезапустите приложение
3. Подождите 2-3 минуты

## 📱 После перезапуска

1. Откройте QR-код: `https://protas090291-otvertka-mob-test-beaf.twc1.net`
2. Metro Bundler предоставляет QR-код на главной странице
3. Отсканируйте QR-код в Expo Go
4. Приложение должно подключиться!

## ✅ Результат

- Metro Bundler работает на порту 8080 (открыт извне)
- Health check доступен через `/status`
- QR-код доступен на главной странице Metro
- Expo Go может подключиться напрямую к Metro на порту 8080
