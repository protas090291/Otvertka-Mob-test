# 🚀 Запуск с любой сети (Wi-Fi и мобильный интернет)

## ✅ Решение проблемы

Теперь приложение работает с **любой сетью** - и через Wi-Fi, и через мобильный интернет!

## 🎯 Быстрый запуск

### Вариант 1: Через скрипт (рекомендуется)
```bash
cd Helper2/mobile-app
./start-tunnel.sh
```

### Вариант 2: Через npm
```bash
cd Helper2/mobile-app
npm run start:tunnel
```

### Вариант 3: Через start-lan.sh (теперь тоже использует tunnel)
```bash
cd Helper2/mobile-app
./start-lan.sh
```

## 📱 Что происходит

1. **Запускается tunnel режим** - создается публичный URL через ngrok
2. **QR код появляется через 1-3 минуты** в файле `expo.log`
3. **Подключение работает** через Wi-Fi и мобильный интернет

## 🔍 Просмотр QR кода

После запуска QR код появится автоматически. Если нет, используйте:

```bash
./show-qr.sh
# или
npm run qr
```

Или откройте файл `expo.log` - там будет QR код и ссылка.

## 🛑 Остановка

```bash
./stop-expo.sh
# или
npm run stop
```

## 📋 Полезные команды

| Команда | Описание |
|---------|----------|
| `./start-tunnel.sh` | Запустить в tunnel режиме |
| `./stop-expo.sh` | Остановить сервер |
| `./show-qr.sh` | Показать QR код |
| `tail -f expo.log` | Смотреть логи в реальном времени |
| `npm run start:tunnel` | То же что `./start-tunnel.sh` |
| `npm run qr` | То же что `./show-qr.sh` |

## ⚠️ Важно

- **Tunnel режим** работает через интернет, поэтому можно переключаться между Wi-Fi и мобильным интернетом
- Первый запуск может занять **1-3 минуты** - это нормально
- Процесс работает в фоне в `screen` сессии - можно закрывать терминал
- Если tunnel оборвется, просто перезапустите: `./start-tunnel.sh`

## 🔧 Если что-то не работает

1. Проверьте что порт 8081 свободен: `lsof -ti:8081`
2. Остановите все процессы: `./stop-expo.sh`
3. Запустите заново: `./start-tunnel.sh`
4. Проверьте логи: `tail -f expo.log`
