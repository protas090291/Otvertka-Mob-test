- ✅ Bucket существует (иначе загрузка не работала бы)

## 🔍 Что нужно проверить:

### Шаг 1: Создайте дефект с фото через приложение

1. Откройте мобильное приложение
2. Перейдите к плану квартиры
3. Создайте новый дефект
4. **Прикрепите фото** (важно!)
5. Сохраните дефект

### Шаг 2: Проверьте в Supabase Dashboard

#### A. Проверьте Storage:
1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Storage → Buckets → defect-photos
3. **Вопрос:** Видите ли вы загруженный файл?
   - ✅ Да - файл загружен
   - ❌ Нет - проблема с загрузкой

#### B. Проверьте базу данных:
1. Table Editor → defects
2. Найдите созданный дефект (поиск по дате создания)
3. Посмотрите на поле `photo_url`
   - **Вопрос:** Есть ли там URL?
   - ✅ Да - URL сохранен
   - ❌ Нет - проблема с сохранением URL

#### C. Проверьте формат URL:
Если URL есть, скопируйте его и проверьте:
- Начинается ли с `https://`? ✅
- Содержит ли `supabase.co`? ✅
- Есть ли `defect-photos` в пути? ✅
- **НЕТ** дублирования `defect-photos/defect-photos/`? ✅

### Шаг 3: Проверьте URL в браузере

1. Скопируйте `photo_url` из базы данных
2. Вставьте в адресную строку браузера
3. Нажмите Enter
4. **Вопрос:** Открывается ли фото?
   - ✅ Да - URL правильный, проблема в приложении
   - ❌ Нет - проблема с URL или RLS политиками

### Шаг 4: Проверьте в приложении

1. Откройте созданный дефект в приложении
2. **Вопрос:** Отображается ли фото?
   - ✅ Да - всё работает!
   - ❌ Нет - проблема с отображением

## 🔧 Возможные проблемы

### Если файл НЕ в Storage:
- Проблема с загрузкой
- Проверьте логи в консоли Expo Go
- Проверьте размер файла (должен быть < 50MB)

### Если URL НЕ в базе данных:
- Проблема с сохранением URL после загрузки
- Проверьте логи в консоли при создании дефекта
- Ищите ошибки после "✅ Фото загружено в Storage!"

### Если URL НЕ открывается в браузере:
- Проблема с форматом URL
- Или проблема с RLS политиками (SELECT)
- Проверьте, публичный ли bucket

### Если URL открывается, но НЕ в приложении:
- Проблема с `expo-image`
- Проблема с кэшированием
- Перезапустите приложение
