## 📋 Настройки для Time Web Cloud

### 1. Команда сборки (Build Command)
```
npm install
```

### 2. Зависимости (Dependencies)
```
npm install
```
*(или оставьте пустым - Time Web Cloud установит автоматически)*

### 3. Команда запуска (Start Command)
```
node server.js
```

**Важно:** Используйте `node server.js` вместо `npx expo start`!

Express сервер (`server.js`) автоматически:
- Запускает Metro Bundler на порту 8081
- Предоставляет health check endpoint на `/status`
- Проксирует запросы к Metro Bundler
- Слушает на порту 8080 (или указанном в переменной PORT)

### 4. Путь проверки состояния (Health Check Path)
```
/status
```

### 5. Переменные окружения (Variables)

Добавьте следующие переменные:

| Переменная | Значение |
|-----------|----------|
| `NODE_ENV` | `production` |
| `PORT` | `8080` |
| `METRO_PORT` | `8081` |
| `EXPO_NO_DOTENV` | `1` |
| `EXPO_DEVTOOLS_LISTEN_ADDRESS` | `0.0.0.0` |
| `NODE_OPTIONS` | `--max-old-space-size=4096` |

## 🔧 После развертывания

### Получение QR-кода

После успешного запуска приложения:

1. Узнайте публичный IP или домен вашего сервера Time Web Cloud
2. Используйте URL в формате:
   ```
   exp://YOUR_SERVER_IP:8081
   ```
   или
   ```
   exp://your-domain.timeweb.cloud:8081
   ```

3. Сгенерируйте QR-код:
   ```
   https://api.qrserver.com/v1/create-qr-code/?size=500x500&data=exp://YOUR_SERVER_IP:8081
   ```

### Проверка работы сервера

Проверьте доступность Express сервера (health check):
```bash
curl http://YOUR_SERVER_IP:8080/status
```

Должен вернуть JSON с статусом `ok`.

Проверьте доступность Metro Bundler через прокси:
```bash
curl http://YOUR_SERVER_IP:8080/
```

Express сервер автоматически проксирует запросы к Metro Bundler на порту 8081.

## 🐛 Устранение проблем

### Если зависимости не устанавливаются

1. Убедитесь что `package.json` обновлен (React 19.2.4)
2. Удалите `package-lock.json` на сервере (если есть)
3. Переразверните приложение

### Если Metro Bundler не запускается

Проверьте логи в панели Time Web Cloud или через SSH:
```bash
tail -f expo.log
```

### Если приложение не доступно извне

1. Проверьте что используется `--host lan`
2. Убедитесь что порт 8081 открыт в firewall
3. Проверьте что порт указан как публичный в настройках сети

## 📱 Подключение через Expo Go

1. Откройте Expo Go на телефоне
2. Отсканируйте QR-код или введите URL вручную:
   ```
   exp://YOUR_SERVER_IP:8081
   ```
3. Приложение должно загрузиться
