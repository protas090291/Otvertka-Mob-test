# 🔧 Решение проблемы обрыва загрузки на 39-40%

## ⚠️ Проблема
При подключении через мобильный интернет по tunnel URL:
- Первая загрузка проходит на 100%
- Вторая загрузка останавливается на 39-40%
- Выдается ошибка "Could not connect to development server"

## ✅ Решение

### Шаг 1: Перезапустите с новыми настройками

```bash
cd Helper2/mobile-app
./stop-expo.sh
./start-tunnel.sh
```

Новые настройки включают:
- Увеличенные таймауты (5 минут)
- Оптимизированные настройки Metro bundler
- Улучшенная стабильность tunnel соединения

### Шаг 2: Дождитесь полной загрузки

После перезапуска подождите 2-3 минуты пока Metro полностью запустится.

### Шаг 3: Пересканируйте QR код

```bash
cd Helper2/mobile-app
screen -r expo-tunnel
```

Отсканируйте QR код заново.

---

## 🔄 Если проблема сохраняется:

### Вариант 1: Используйте HTTPS tunnel

Tunnel URL должен использовать HTTPS вместо HTTP. Проверьте:

```bash
cd Helper2/mobile-app
./get-tunnel-url.sh
```

URL должен быть: `exp://xxxx-xxxx.exp.direct` (без указания порта)

### Вариант 2: Очистите кэш и перезапустите

```bash
cd Helper2/mobile-app
./stop-expo.sh
rm -rf node_modules/.cache
rm -rf .expo
npx expo start --tunnel --clear
```

### Вариант 3: Уменьшите размер bundle

Проблема может быть в большом размере bundle. Попробуйте:

1. Закройте другие приложения на телефоне
2. Используйте Wi-Fi вместо мобильного интернета для первой загрузки
3. После первой загрузки можно переключаться на мобильный интернет
