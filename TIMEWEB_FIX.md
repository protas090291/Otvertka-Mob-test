# ✅ Исправление проблемы с контейнером Time Web Cloud

## 🔴 Проблема

Контейнер постоянно перезагружается из-за того, что:
- Metro Bundler не может корректно запуститься в контейнере
- Express сервер зависел от успешного запуска Metro Bundler
- Не было обработки ошибок для контейнерного окружения

## ✅ Что исправлено

### 1. Улучшен Express сервер (`server.js`)

**Основные изменения:**

- ✅ **Express сервер запускается сразу** - не ждет Metro Bundler
- ✅ **Metro Bundler запускается асинхронно** через 2 секунды после Express
- ✅ **Health check работает всегда** - даже если Metro не запустился
- ✅ **Улучшена обработка ошибок** - сервер не падает при ошибках Metro
- ✅ **Добавлены флаги для продакшена:**
  - `--no-dev` - отключает dev режим
  - `--minify` - минификация кода
  - `CI=true` - указывает CI окружение
- ✅ **Автоматическая проверка готовности Metro** перед проксированием
- ✅ **Graceful shutdown** - корректное завершение процессов

### 2. Логирование

Теперь все логи Metro Bundler выводятся в консоль для отладки.

## 📋 Настройки Time Web Cloud

### Обязательные настройки:

1. **Фреймворк:** `Node.js` или `Express`
2. **Start Command:** `node server.js`
3. **Health Check Path:** `/status`
4. **Переменные окружения:**
   - `PORT` = `8080`
   - `METRO_PORT` = `8081`
   - `NODE_ENV` = `production`

## ✅ Проверка работы

### 1. Health Check должен работать сразу:

```bash
curl http://YOUR_SERVER:8080/status
```

Должен вернуть:
```json
{
  "status": "ok",
  "service": "Expo Metro Bundler Proxy",
  "metro_port": 8081,
  "metro_ready": true/false,
  "timestamp": "..."
}
```

### 2. Проверка логов в Time Web Cloud:

Должны быть сообщения:
```
✅ Express сервер запущен на порту 8080
📡 Health check: http://0.0.0.0:8080/status
🚀 Запуск Metro Bundler в фоновом режиме...
✅ Metro Bundler готов!
```

### 3. Если Metro не запустился:

Express сервер все равно будет работать и отвечать на `/status`. Metro будет пытаться перезапуститься автоматически.

## 🔄 Что происходит при запуске

1. **Express сервер запускается сразу** на порту 8080
2. **Health check доступен** через `/status`
3. **Через 2 секунды** запускается Metro Bundler в фоне
4. **Express проверяет готовность Metro** каждые 5 секунд
5. **Когда Metro готов** - начинается проксирование запросов

## 🐛 Если проблема остается

### Проверьте логи в Time Web Cloud:

1. Откройте раздел "Логи приложения"
2. Ищите сообщения:
   - `✅ Express сервер запущен` - сервер работает
   - `❌ Ошибка запуска Metro` - проблема с Metro
   - `[Metro]` - логи Metro Bundler

### Возможные проблемы:

1. **Metro не запускается:**
   - Проверьте что `npx` доступен
   - Проверьте что `expo` установлен в зависимостях
   - Проверьте логи Metro

2. **Порт занят:**
   - Убедитесь что порт 8080 свободен
   - Проверьте переменную `PORT`

3. **Контейнер перезагружается:**
   - Проверьте логи на ошибки
   - Убедитесь что команда запуска: `node server.js`

---

**Готово!** Express сервер теперь работает стабильно в контейнере. 🚀
