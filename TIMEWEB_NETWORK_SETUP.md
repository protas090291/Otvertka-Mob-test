# 🔧 Настройка сети Time Web Cloud для Expo Go

## ⚠️ Проблема: "Could not connect to server"

Если Expo Go показывает ошибку "Could not connect to server", это означает что порт 8081 не доступен извне.

## ✅ Решение: Открыть порт 8081

### Шаг 1: Откройте настройки сети в Time Web Cloud

1. Войдите в панель Time Web Cloud
2. Откройте ваше приложение `otvertka-mob`
3. Перейдите в раздел **"Сеть"** или **"Network"**

### Шаг 2: Добавьте порт 8081

1. Найдите раздел **"Порты"** или **"Ports"**
2. Добавьте новый порт:
   - **Порт:** `8081`
   - **Протокол:** `TCP`
   - **Тип:** `Публичный` или `Public`
   - **Описание:** `Metro Bundler для Expo Go`

### Шаг 3: Сохраните и перезапустите

1. Сохраните изменения
2. Перезапустите приложение
3. Подождите 1-2 минуты для применения изменений

## 🔍 Проверка доступности порта

После настройки проверьте доступность порта:

```bash
curl http://protas090291-otvertka-mob-test-beaf.twc1.net:8081/status
```

Должен вернуть ответ от Metro Bundler.

## 📱 Альтернативное решение: Использовать Express прокси

Если порт 8081 нельзя открыть, можно использовать Express сервер как прокси:

1. В Expo Go используйте URL:
   ```
   exp://protas090291-otvertka-mob-test-beaf.twc1.net:8080
   ```
   (порт 8080 вместо 8081)

2. Express сервер автоматически проксирует запросы к Metro Bundler

**Но это требует дополнительной настройки Metro для работы через прокси.**

## ✅ Рекомендуемое решение

**Лучше всего открыть порт 8081** в настройках сети Time Web Cloud, так как:
- Expo Go требует прямой доступ к Metro Bundler
- Это стандартный способ подключения
- Не требует дополнительных настроек

---

**После открытия порта 8081 пересканируйте QR-код в Expo Go!**
