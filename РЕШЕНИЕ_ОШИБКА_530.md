# 🔧 Решение ошибки HTTP 530 Cloudflare Tunnel

## ⚠️ Проблема:

Ошибка **HTTP 530** от Cloudflare Tunnel означает:
- ❌ Cloudflare Tunnel не может подключиться к Metro Bundler
- ❌ Metro Bundler не запущен или не отвечает на `localhost:8081`
- ❌ Проблема синхронизации запуска процессов

---

## ✅ Решение:

### Причина ошибки:

Cloudflare Tunnel запускается **до того** как Metro Bundler полностью готов, и не может подключиться к `localhost:8081`.

### Что исправлено:

1. **Увеличено время ожидания Metro** - с 30 до 60 попыток (2 минуты)
2. **Добавлена дополнительная пауза** после готовности Metro (3 секунды)
3. **Улучшена синхронизация** запуска процессов
4. **Добавлены проверки статуса** перед запуском Cloudflare Tunnel

---

## 🚀 Использование:

### Автоматический перезапуск:

```bash
cd Helper2/mobile-app
./stop-expo.sh
./start-cloudflare-stable.sh
```

Или через npm:
```bash
npm run stop
npm run start:cloudflare
```

---

## 📋 Проверка статуса:

### Проверить Metro Bundler:

```bash
curl http://localhost:8081/status
```

Должен вернуть статус 200.

### Проверить процессы:

```bash
# Metro Bundler
ps aux | grep "expo start"

# Cloudflare Tunnel
ps aux | grep cloudflared

# Все screen сессии
screen -ls
```

### Проверить логи:

```bash
# Логи Metro
tail -f expo.log

# Логи Cloudflare Tunnel
tail -f cloudflare.log

# Логи keep-alive
tail -f keep-alive.log
```

## 🔍 Если проблема остается:

### Вариант 1: Ручной запуск с задержкой

```bash
cd Helper2/mobile-app

# 1. Запустите Metro вручную
npx expo start --clear

# 2. Подождите пока увидите "Metro waiting on..."
# 3. В другом терминале запустите Cloudflare Tunnel
cloudflared tunnel --url http://localhost:8081
```

### Вариант 2: Проверьте порт 8081

```bash
# Проверьте что порт свободен
lsof -i :8081

# Если занят, освободите:
kill -9 $(lsof -t -i:8081)
```

### Вариант 3: Используйте Ngrok Pro

Если Cloudflare Tunnel продолжает вызывать проблемы:

```bash
npm run start:ngrok-pro
```

Ngrok Pro более стабилен и лучше синхронизируется с Metro.
