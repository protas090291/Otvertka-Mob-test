# 🔧 Исправление ошибки канала "production"

## ⚠️ Ошибка:

```
There is no channel named production for the app with ID: bce5f246-91b5-491f-bcf1-28df1ccf3c21
```

## 🔍 Причина:

EAS Update использует **branches**, а не **channels**. В URL использовался `channel-name=production`, но нужно использовать `branch`.

---

## ✅ Решение:

### Правильный формат QR кода:

**❌ Неправильно (с channel):**
```
https://qr.expo.dev/eas-update?projectId=...&channel=production
```

**✅ Правильно (с branch):**
```
https://qr.expo.dev/eas-update?projectId=bce5f246-91b5-491f-bcf1-28df1ccf3c21&runtimeVersion=2.0.0&branch=production
```

---

## 📱 Новый QR код:

QR код создан с правильным форматом и открыт в браузере!

**Попробуйте отсканировать его в Expo Go - должно работать!**

## 🔧 Если все еще не работает:

### Вариант 1: Проверьте что branch существует

```bash
cd Helper2/mobile-app
npx eas-cli update:list --branch production --non-interactive
```

### Вариант 2: Опубликуйте обновление заново

```bash
cd Helper2/mobile-app
npx eas-cli update --branch production --message "Update"
```

### Вариант 3: Используйте Cloudflare Tunnel

Если EAS Update не работает с Expo Go, используйте:
```bash
npm run start:cloudflare
```

- **Branches** используются для EAS Update
- **Channels** - это устаревший формат
- Используйте `branch=production` в URL
