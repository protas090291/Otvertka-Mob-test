# 🔄 Keep-Alive Решение для проблемы неактивности

## ⚠️ Проблема:

После того как телефон выключен 10+ минут:
- ❌ Нужно пересканировать QR код
- ❌ Дефекты не загружаются
- ❌ Новые дефекты не создаются

**Причина:** Tunnel/Metro соединение истекает после периода неактивности.

---

## ✅ Решение: Keep-Alive механизм

Добавлен автоматический keep-alive механизм, который:
- ✅ Поддерживает Metro Bundler соединение активным
- ✅ Предотвращает истечение после неактивности
- ✅ Работает автоматически при запуске Cloudflare Tunnel
- ✅ Не требует дополнительных действий

---

## 🚀 Как использовать:

### Автоматический запуск (рекомендуется):

Keep-alive запускается автоматически при запуске Cloudflare Tunnel:

```bash
cd Helper2/mobile-app
npm run start:cloudflare
```

Keep-alive будет работать автоматически в фоне.

---

### Ручной запуск (если нужно):

Если нужно запустить keep-alive отдельно:

```bash
cd Helper2/mobile-app
./keep-alive.sh
```

---

## 📋 Управление:

### Остановить все процессы (включая keep-alive):

```bash
./stop-expo.sh
```

### Проверить статус keep-alive:

```bash
screen -ls | grep keepalive
```

### Посмотреть логи keep-alive:

```bash
tail -f keep-alive.log
```

---

## 💡 Как это работает:

1. **Keep-alive скрипт** запускается в фоне в `screen` сессии
2. **Каждые 60 секунд** делает легкий запрос к Metro Bundler (`/status`)
3. **Поддерживает соединение активным** - предотвращает истечение
4. **Работает автоматически** - не требует вмешательства

---

## 🔧 Технические детали:

- **Интервал проверки:** 60 секунд
- **Запрос:** `GET http://localhost:8081/status`
- **Процесс:** Запущен в `screen` сессии `expo-keepalive`
- **Логи:** Сохраняются в `keep-alive.log`

---

## ✅ Ожидаемый результат:

После добавления keep-alive:
- ✅ Соединение не будет истекать после неактивности
- ✅ Не нужно пересканировать QR код после паузы
- ✅ Дефекты будут загружаться нормально
- ✅ Создание новых дефектов будет работать

---

## ⚠️ Важно:

1. **Keep-alive работает только если Metro запущен**
   - Если Metro остановился, keep-alive не поможет
   - Используйте автоматический перезапуск Metro (уже настроен)

2. **Keep-alive не решает проблемы с переключением сетей**
   - Если переключили сеть, нужно пересканировать QR код
   - Это нормальное поведение Expo Go

3. **Keep-alive работает только для Metro**
   - Tunnel соединение также может истекать
   - Cloudflare Tunnel обычно более стабилен чем ngrok
