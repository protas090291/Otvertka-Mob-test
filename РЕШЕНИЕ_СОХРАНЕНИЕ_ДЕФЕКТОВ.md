# 🔧 Решение проблемы сохранения дефектов через tunnel

## ⚠️ Проблема:

Приложение работает, данные читаются, но операции сохранения (создание/обновление дефектов) не работают через tunnel при переключении на мобильный интернет.

## 🔍 Причина:

1. **Таймауты слишком короткие** - операции сохранения занимают больше времени через tunnel
2. **Нестабильное соединение** - tunnel может обрываться во время запросов
3. **Нет retry логики** - при ошибке запрос не повторяется

---

## ✅ Решение применено:

### 1. Увеличены таймауты Supabase

**Было:** 30 секунд  
**Стало:** 2 минуты (120 секунд)

Это дает больше времени для операций сохранения через нестабильное tunnel соединение.

### 2. Добавлена retry логика

**Для `createDefect`:**
- 3 попытки создания дефекта
- Пауза 2 секунды между попытками
- Автоматический retry при ошибках

**Для `updateDefect`:**
- 3 попытки обновления дефекта
- Пауза 2 секунды между попытками
- Автоматический retry при ошибках

### 3. Улучшена обработка ошибок

- Детальное логирование ошибок
- Информация о количестве попыток
- Понятные сообщения об ошибках

---

## 📋 Что делать:

### Шаг 1: Перезапустите приложение

```bash
cd Helper2/mobile-app
./stop-expo.sh
npm run start:tunnel:stable
npm run qr:stable
```

### Шаг 2: Отсканируйте QR код заново

Это применит новые настройки.

### Шаг 3: Попробуйте создать дефект

Теперь операции сохранения должны работать стабильнее.

---

## 💡 Как это работает:

1. **При создании дефекта:**
   - Первая попытка отправки запроса
   - Если ошибка - ждет 2 секунды и повторяет
   - До 3 попыток всего
   - Если все попытки неудачны - показывает ошибку

2. **При обновлении дефекта:**
   - Та же логика retry
   - Автоматические повторы при ошибках
   - Больше времени на выполнение

## 🔍 Если проблема остается:

### Проверьте логи:

```bash
# В Expo Go откройте DevTools и проверьте консоль
# Или проверьте логи Metro:
tail -f expo.log
```

### Возможные причины:

1. **Tunnel обрывается** - используйте Ngrok Pro
2. **Слишком большой запрос** - проверьте размер фото
3. **Проблемы с Supabase** - проверьте доступность Supabase

---

## 🎯 Рекомендация:

Если проблема остается после применения исправлений:

1. **Используйте Ngrok Pro** для стабильного tunnel:
   ```bash
   ./start-ngrok-pro-for-network-switch.sh
   ```

2. **Проверьте размер фото** - большие фото могут вызывать проблемы

3. **Проверьте интернет соединение** - убедитесь что оно стабильное
