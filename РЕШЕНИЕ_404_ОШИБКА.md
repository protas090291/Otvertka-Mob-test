# 🔧 Решение ошибки 404 в Expo Go

## ⚠️ Проблема:

Ошибка "Could not connect to development server" с URL типа:
```
http://relktho-zakharprotas-8081.exp.direct/...
```

## ✅ Решение:

### 1. Используйте правильный протокол:

**❌ Неправильно:**
```
http://relktho-zakharprotas-8081.exp.direct
```

**✅ Правильно:**
```
exp://relktho-zakharprotas-8081.exp.direct
```

### 2. Убедитесь что Metro bundler запущен:

```bash
cd Helper2/mobile-app
curl http://localhost:8081/status
```

Должен вернуть статус Metro.

### 3. Проверьте tunnel:

```bash
curl http://localhost:4040/api/tunnels
```

Должен показать активный tunnel.

### 4. Перезапустите tunnel:

```bash
cd Helper2/mobile-app
./stop-expo.sh
npm run start:tunnel:stable
npm run qr:stable
```

---

## 📱 Правильный QR код:

QR код должен содержать URL с протоколом `exp://`:

```
exp://relktho-zakharprotas-8081.exp.direct
```

**НЕ используйте `http://` в Expo Go!**

---

## 🔍 Проверка:

1. **Metro работает:** `curl http://localhost:8081/status`
2. **Tunnel активен:** `curl http://localhost:4040/api/tunnels`
3. **QR код правильный:** содержит `exp://` протокол

---

## 💡 Важно:

- Expo Go требует протокол `exp://`, а не `http://`
- Metro bundler должен быть запущен и доступен
- Tunnel должен быть активен и подключен
