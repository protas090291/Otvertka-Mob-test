# 📱 Как получить QR код для тестирования

## Проблема
При фоновом запуске через `nohup` QR код не всегда появляется в логах, так как Expo выводит его только в интерактивном режиме.

## ✅ Решение 1: Запуск в отдельном терминале (Рекомендуется)

### Для Tunnel режима (работает через интернет):
```bash
cd Helper2/mobile-app
npx expo start --tunnel --clear
```

### Для LAN режима (только локальная сеть, быстрее):
```bash
cd Helper2/mobile-app
npx expo start --lan --clear
```

**QR код появится в терминале через 1-2 минуты** после запуска Metro Bundler.

## ✅ Решение 2: Использование screen (macOS/Linux)

Если у вас установлен `screen`:

```bash
# Установить screen (если нет)
# macOS: brew install screen

# Запустить в screen сессии
cd Helper2/mobile-app
screen -S expo npx expo start --tunnel --clear

# Отключиться от сессии: Ctrl+A, затем D
# Вернуться к сессии: screen -r expo
# Посмотреть все сессии: screen -ls
```

## ✅ Решение 3: Фоновый запуск + проверка через браузер

1. Запустите в фоне:
```bash
cd Helper2/mobile-app
./start-background.sh
```

2. Подождите 2-3 минуты

3. Откройте в браузере: `http://localhost:8081`

4. Или проверьте через API:
```bash
curl http://localhost:8081/status
```

## 📋 Проверка статуса

```bash
# Проверить работает ли процесс
ps aux | grep "expo start"

# Проверить логи
tail -f expo.log

# Проверить порт
lsof -ti:8081
```

## 💡 Рекомендация

**Лучший способ для тестирования:**
1. Откройте отдельный терминал
2. Выполните: `cd Helper2/mobile-app && npx expo start --tunnel --clear`
3. Дождитесь появления QR кода (1-2 минуты)
4. Отправьте QR код тестировщикам
5. Оставьте терминал открытым (или используйте screen)

QR код будет виден в терминале и его можно будет отправить тестировщикам.
