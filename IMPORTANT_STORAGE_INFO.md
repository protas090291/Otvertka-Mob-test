# ✅ Важная информация о диагностике Storage

## Почему bucket может не находиться?

**Это нормально!** Anon key (публичный ключ) часто не имеет прав на `listBuckets()`, но это **не означает**, что bucket не существует или не работает.

### Что это значит:
- ✅ Bucket существует (иначе загрузка не работала бы)
- ✅ Загрузка работает (вы подтвердили, что тестовые файлы загрузились)
- ⚠️ Anon key не может получить список bucket'ов (это ограничение прав доступа)

### Как проверить bucket вручную:
1. Откройте Supabase Dashboard
2. Storage → Buckets
3. Найдите `defect-photos` - он должен быть там!

## 🔍 Что проверить дальше

### 1. Проверка загруженных файлов

После загрузки тестового файла:
1. Откройте Supabase Dashboard
2. Storage → defect-photos
3. Посмотрите список файлов
4. Проверьте, что файлы действительно там

### 2. Проверка URL после загрузки

Когда файл загружается, код должен:
1. Загрузить файл в Storage ✅ (работает)
2. Получить URL файла (signed URL или public URL)
3. Сохранить URL в базе данных в поле `photo_url` таблицы `defects`

**Проверьте в базе данных:**
- Откройте Supabase Dashboard
- Table Editor → defects
- Найдите дефект, к которому прикрепляли фото
- Проверьте поле `photo_url` - есть ли там URL?

### 3. Проверка формата URL

URL должен быть в формате:
- `https://yytqmdanfcwfqfqruvta.supabase.co/storage/v1/object/sign/defect-photos/...` (signed URL)
- ИЛИ `https://yytqmdanfcwfqfqruvta.supabase.co/storage/v1/object/public/defect-photos/...` (public URL)

**НЕ должно быть:**
- `defect-photos/defect-photos/...` (дублирование пути)
- Относительных путей без домена

### 4. Проверка отображения фото

Если URL правильный, но фото не отображается:
1. Скопируйте URL из базы данных
2. Откройте его в браузере
3. Если открывается - проблема в приложении
4. Если не открывается - проблема в Storage/RLS

## 🔧 Возможные проблемы и решения

### Проблема: Файл загружается, но URL не сохраняется в базе

**Причина:** Ошибка при обновлении дефекта после загрузки фото

**Решение:** Проверьте логи в консоли при создании дефекта

### Проблема: URL сохраняется, но фото не отображается

**Причина 1:** URL неправильного формата
- Проверьте формат URL в базе данных
- Убедитесь, что это полный URL с доменом

**Причина 2:** RLS политики не позволяют читать файл
- Проверьте RLS политики для SELECT
- Или сделайте bucket публичным

**Причина 3:** Signed URL истек
- Signed URL действителен 1 год (в коде)
- Если истек, нужно создать новый

### Проблема: Фото отображается в браузере, но не в приложении

**Причина:** Проблема с `expo-image` или кэшированием

**Решение:**
1. Перезапустите приложение
2. Очистите кэш Expo
3. Проверьте логи в консоли Expo Go
