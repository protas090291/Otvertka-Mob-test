# ⚙️ Настройки Time Web Cloud - Краткая инструкция

## 🔴 КРИТИЧЕСКИ ВАЖНО: Исправление ошибки "Несоответствие фреймворка"

### Проблема
Time Web Cloud показывает ошибку: "Несоответствие между фреймворком и командой запуска"

### Решение

#### 1. Выбор фреймворка в панели Time Web Cloud

В настройках вашего приложения:

1. Откройте настройки приложения в Time Web Cloud
2. Найдите раздел **"Фреймворк"** или **"Framework"**
3. Выберите: **`Node.js`** или **`Express`**
   - ❌ НЕ выбирайте "React Native" или "Expo"
   - ✅ Выберите "Node.js" или "Express"

#### 2. Команда запуска (Start Command)

Убедитесь, что команда запуска установлена как:

```
node server.js
```

**НЕ используйте:**
- ❌ `npx expo start`
- ❌ `npm start` (если он указывает на expo start)
- ❌ `expo start`

#### 3. Health Check Path

Установите путь проверки состояния:

```
/status
```

#### 4. Переменные окружения

Добавьте следующие переменные:

| Переменная | Значение |
|-----------|----------|
| `PORT` | `8080` |
| `METRO_PORT` | `8081` |
| `NODE_ENV` | `production` |

#### 5. После настройки

1. Сохраните изменения
2. Перезапустите приложение
3. Проверьте логи - должно быть:
   ```
   ✅ Express сервер запущен на порту 8080
   📡 Health check: http://0.0.0.0:8080/status
   🔗 Metro Bundler будет доступен на порту 8081
   ```

## ✅ Проверка работы

После перезапуска проверьте:

```bash
curl http://YOUR_SERVER_IP:8080/status
```

Должен вернуть:
```json
{
  "status": "ok",
  "service": "Expo Metro Bundler Proxy",
  "metro_port": 8081,
  "timestamp": "..."
}
```

## 🔧 Исправление уязвимостей (опционально)

После успешного деплоя, подключитесь к серверу по SSH и выполните:

```bash
npm audit fix
```

Это обновит безопасные версии зависимостей без breaking changes.
