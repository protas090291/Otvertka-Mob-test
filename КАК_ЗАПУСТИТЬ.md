# 🚀 Как запустить приложение и увидеть QR-код

## Проблема: Не видно QR-код

Если QR-код не появляется, выполните следующие шаги:

## ✅ Решение 1: Остановить старые процессы и запустить заново

### В терминале выполните:

```bash
# Перейдите в папку приложения
cd Helper2/mobile-app

# Остановите все процессы Expo
lsof -ti:8081 | xargs kill -9 2>/dev/null
lsof -ti:8082 | xargs kill -9 2>/dev/null

# Запустите с очисткой кеша
npx expo start --clear
```

## ✅ Решение 2: Использовать скрипт запуска

```bash
cd Helper2/mobile-app
./start.sh
```

Или если скрипт не работает:

```bash
chmod +x start.sh
./start.sh
```

## ✅ Решение 3: Ручной запуск с другим портом

```bash
cd Helper2/mobile-app
npx expo start --clear --port 8082
```

## 🔍 Проверка проблем:

### 1. Проверьте, что вы в правильной папке:
```bash
pwd
# Должно быть: .../Helper2/mobile-app
```

### 2. Проверьте наличие node_modules:
```bash
ls node_modules | head -5
# Должны быть папки: expo, react, react-native и т.д.
```

### 3. Если node_modules нет, установите:
```bash
npm install
```

### 4. Проверьте версию Node.js:
```bash
node --version
# Должна быть 18 или выше
```

## ⚠️ Если ничего не помогает:

1. **Полная переустановка:**
```bash
cd Helper2/mobile-app
rm -rf node_modules
rm -rf .expo
npm install
npx expo start --clear
```

2. **Проверьте, что Expo CLI установлен:**
```bash
npm install -g expo-cli
```

3. **Запустите в веб-режиме для проверки:**
```bash
npx expo start --web
```

## 📞 Дополнительная помощь:

Если проблема сохраняется, проверьте:
- Интернет-соединение (Expo нужен интернет)
- Файрвол не блокирует порт 8081
- Другие приложения не используют порт 8081
