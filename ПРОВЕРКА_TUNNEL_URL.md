# 🔍 Проверка Tunnel URL

## ⚠️ Важно: URL должен содержать tunnel адрес!

Для работы через разные сети (Wi-Fi и мобильный интернет) QR код **ДОЛЖЕН** содержать:

### ✅ Правильный URL (работает с любой сетью):
```
exp://xxxx-xxxx.exp.direct:80
```
или
```
exp://xxxx-xxxx.anonymous.exp.direct:80
```

### ❌ Неправильный URL (работает только в одной сети):
```
exp://192.168.x.x:8081
exp://localhost:8081
exp://10.x.x.x:8081
```

---

## 🔍 Как проверить ваш QR код:

### Способ 1: Подключитесь к screen сессии
```bash
cd Helper2/mobile-app
screen -r expo-tunnel
```

Посмотрите на URL в QR коде - он должен содержать `exp.direct` или `ngrok`.

### Способ 2: Проверьте логи
```bash
cd Helper2/mobile-app
tail -200 expo.log | grep -E "exp://|expo://"
```

URL должен содержать `exp.direct`.

---

## 🚨 Если URL неправильный (локальный адрес):

### Решение: Перезапустите с явным указанием tunnel

```bash
cd Helper2/mobile-app
./stop-expo.sh
npx expo start --tunnel --clear
```

Подождите 2-3 минуты и проверьте QR код снова.

---

## ✅ Если URL правильный, но все равно не работает:

1. **Проверьте что tunnel действительно работает:**
   ```bash
   ps aux | grep ngrok
   ```
   Должен быть процесс ngrok.

2. **Проверьте что вы используете tunnel режим:**
   ```bash
   ps aux | grep "expo start"
   ```
   Должно быть `--tunnel` в команде.

3. **Перезапустите tunnel:**
   ```bash
   ./stop-expo.sh
   ./start-tunnel.sh
   ```

---

## 💡 Важно:

- **Tunnel URL** (`exp.direct`) работает через интернет - можно переключаться между сетями
- **Локальный URL** (`192.168.x.x`) работает только в одной Wi-Fi сети
- При переключении на мобильный интернет локальный URL перестанет работать

---

**Проверьте ваш QR код и убедитесь что URL содержит `exp.direct`!**
